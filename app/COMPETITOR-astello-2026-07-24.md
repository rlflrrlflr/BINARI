## 2. 온보딩 (5단계, 초개인화 대화형)

1. **이름** — "별비가 부를 이름 / 이름이 어떻게 되세요?"(별명 허용) ※ 마스코트를 밤비/별비 혼용 — 네이밍 일관성 흔들림(약점)
2. **생년월일**(양력) — "강석우님, 틀리게 적으면 결과도 달라져요"
3. **태어난 시간**(생시, 0~23시) — "생시를 잘 모르겠어요" 폴백 링크
4. **태어난 도시** — "예: 서울, 부산, 뉴욕", "태어난 곳을 잘 모르겠어요" 폴백

→ 매 단계 **이름 호명 + 신뢰 유도 문구**("거의 다 왔어요, 강석우님" / "마지막이에요")로 이탈 방지. 카피 완성도 높음.

## 3. 구조·맥락수집·로딩

- 하단 탭: **홈 / 리딩 / 무료질문(중앙 CTA) / 대화 / 마이페이지**
- **맥락 수집 단계**(온보딩 후): "지금 어떤 일과 어떤 관계를 가지고 있어?"
  - 직업: 학생·직장인·주부·사업가·프리랜서·쉬는 중·퇴직
  - 관계: 연애 중·솔로·결혼 생활 중·이혼 → **실제 삶의 맥락으로 리딩 개인화** (비나리는 가치관은 받지만 현재 상황은 안 받음)
- **로딩 화면 = 신뢰·가치 전환 장치**: 밤비가 명상하며 "강석우 하늘을 읽고 있어 / 잠깐만 기다려줘 ✨" + 단계별 진행(🔭행성 위치 계산 → ⚡하우스 배치 분석 → 📖미리보기 생성 → ✨마무리) + **별점 후기**("김\*\* — 쓰레드에서 우연히...대박터졌다는 글 보고" = Threads 바이럴 사회적 증명). *대기시간을 죽은 시간이 아니라 '정성 들여 계산 중'으로 연출.*
- **대화(챗봇)**: 라이트 테마 채팅, 별비가 **선제 반말 인사**("석우아, 나른한 오후이네. 요즘 어떤 게 제일 신경 쓰여?") → 지속 상호작용. 질문 300자 제한.

## 3-2. 💰 수익 구조 (영상에서 확인 — 핵심)

**대화 = 회당 과금(크레딧/충전) 모델.** 채팅 상단 "**0회 남음**" 카운터 → 소진 시 "**✦ 대화 충전하기**":

| 패키지 | 가격 | 회당 단가 |
|---|---|---|
| 1회 | 990원 | 990원 |
| 3회 | 2,670원 | 890원 |
| **10회** | **7,900원** | **790원 (BEST)** |
| 50회 | 34,500원 | 690원 |

"많이 살수록 회당 단가 ↓ · 결제 후 즉시 잔액 반영 · **유효기간 없음**"

> ⚠️ **전략적 핵심**: 이건 비나리가 뺐던 **복채(질문당 과금)** 모델 그 자체다. 최근접 경쟁자가 **광고비를 태우며 라이브로** 이 모델을 굴리고 있음 = 모델은 시장에서 검증됨. 비나리가 뺀 건 모델이 틀려서가 아니라 **"복채/1900원"이라는 프레이밍**이 뜬금없었기 때문. astello는 "**대화 충전**(회당 990원, 볼륨 할인, 무기한)"으로 자연스럽게 포장 — 미신적 '복채'가 아니라 '크레딧'. 비나리도 판결/심층상담을 **크레딧 충전**으로 재설계할 여지가 크다(무료 N회 → 충전).

## 3-3. 네이밍 불일치 (약점)

## 6. 시사점

### astello가 앞서는 것 (따라잡을 것)
1. **초개인화 온보딩 카피** — 매 단계 이름 호명·"틀리게 적으면 결과 달라져요"류 신뢰 유도. 비나리 온보딩도 이 온기·개인화 필요.
2. **가동 중인 배포 엔진** — 인스타 링크인바이오 + 메타광고 + 카톡 웹뷰. 비나리는 프로덕트는 앞서는데 "유저를 어떻게 데려오나"가 공백.
3. **명확한 수익 깔때기** — 무료질문 훅 → 프리미엄. 비나리는 복채 제거 후 수익모델 미정.
4. **마스코트 = 리텐션·친밀·확장** — 귀여운 캐릭터로 재방문·굿즈·이모티콘. (비나리가 정령을 넣었다 뺀 부분 — 재방문 훅 관점 재고 여지.)
5. **대화 지속성** — "뭐든 물어봐"로 계속 말 걸 여지. 비나리 판결은 1회성에 가까움.

### 비나리가 이길 수 있는 것 (파고들 것)
1. **결정을 준다** — astello=정보, 비나리=결단. 망설이는 유저에겐 단언이 압도적으로 강력. **핵심 차별점**(레드팀 감사와 일치).
2. **비주얼 임팩트 = 공유 무기** — astello는 정적 일러스트, 비나리는 살아있는 파티클. 방금 고도화한 FBO 엔진으로 "내 사주가 이런 수호신이 됐다" 영상 = **astello의 배포채널(인스타)을 더 강한 비주얼로 역공** 가능.
3. **사주별 유니크** — astello 마스코트는 예쁘지만 만인 공통. 비나리 수호신은 사주 파라미터로 매번 다른 형태 = "나만의 것".
4. **진입 마찰** — astello는 태어난 도시까지(4단계, 점성술 하우스). 비나리 사주는 더 가벼움.

### 실제 반영 후보 (우선순위)
1. **온보딩 초개인화 카피** — 이름 호명·신뢰 유도(astello 벤치마크). *가장 빠른 개선.*
2. **인스타/메타 배포 퍼널** — 수호신 생성 영상을 공유 자산화 → 링크인바이오. 비주얼 우위를 배포로 전환.
3. **수익모델 재설계 — 최우선 재고** — astello가 **회당 크레딧 과금(990~690원, 무기한)**으로 라이브 검증한 모델. 비나리가 뺀 복채가 바로 이것. "복채/1900원" 프레이밍만 버리고 "**심층 판결/재상담 충전**(무료 N회 → 크레딧)"으로 재포장 검토. *과금은 SYS 판결 프롬프트와 무관하므로 채점 동결과 별개로 진행 가능.*
4. **재방문 훅 재고** — 마스코트/정령 요소(명상 무드 안 깨는 선에서).
5. **포지셔닝 각인** — "우리는 정보가 아니라 **결정**을 준다."
