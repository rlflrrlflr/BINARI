# 비나리(BINARI) 개발 인계서 — v15

> **한 줄 요약:** 생년월일·성격 데이터로 '수호신'을 조립하고, 사용자의 고민에 AI가 판결을 내리는 의사결정 지원 웹앱. 현재 v15까지 코드 완성, **문법·시각 로직은 검증 완료 / 실제 API 동작은 미검증** 상태.
>
> **작성 기준일:** 2026-07-10 · **최신 버전:** `비나리_비주얼프로토타입_MVP_v15.jsx` · **형태:** React 웹앱(단일 JSX 아티팩트)

---

## 1. 제품 한눈에

- **정체:** 브라우저에서 도는 **웹앱**(설치형 PC 앱 아님). `.jsx` = 소스코드라 빌드/아티팩트 환경을 거쳐야 실행됨.
- **세계관:** "수호신은 사라진 게 아니라 흩어졌다" — 흩어진 조각(생일·성격 데이터)을 모아 나만의 수호신을 복원. **"수호신은 이미 너를 안다"** 가 핵심 제약(= 자동으로 뽑을 수 있는 값은 손으로 되묻지 않는다).
- **흐름:** 온보딩(생일·MBTI·혈액형 입력) → 파라메트릭 수호신 생성 → 질문 → AI 판결(3층 공개).

---

## 2. 현재 상태 (v15에 든 것)

### 2-1. 판결 엔진 — 2콜 구조
- **콜1 (결론만):** `max_tokens: 320`. verdict + GO/STOP/HOLD만 빠르게 출력 → **L1 즉시 노출**.
- **콜2 (근거만):** `max_tokens: 1500`. 콜1의 verdict를 "확정, 뒤집기 금지"로 넘겨 근거만 생성 → **결론과 근거가 구조적으로 절대 안 어긋남**. 백그라운드로 미리 로드.
- 프롬프트 캐싱: 시스템프롬프트+고정 프로필을 `system`에 담고 `cache_control: ephemeral`.
- 대화 기억: 최근 6턴(12메시지) 누적 → "이어묻기" 지원.

### 2-2. 리빌 UI — 3층
| 층 | 공개 방식 | 내용 | 출처 |
|---|---|---|---|
| L1 결론 | 즉시 | verdict + 방향(GO/STOP/HOLD) | 콜1 |
| L2 왜? | "왜 이렇게 봤어?" 클릭 | 수호신 한 줄 + 접전 pips + 정령 한마디 | 콜2 |
| L3 세부 | 카드 뒤집기 | 지표별 근거 표 + 면책 | 콜2 |

→ 클릭이 콜2 지연을 가리는 "시간 벌이" 역할.

### 2-3. 수호신 비주얼 — 지표별 독립 시각축
| 지표 | 시각 축 |
|---|---|
| 사주 주기운(오행) | 형태(불·물·나무·빛살·구) |
| 별자리 | 주색 hue 회전(이전엔 안 쓰이던 죽은 값) |
| 사주 오행 분포 2순위 | 강조색(입자 24% 혼합) |
| 수비학 라이프패스 | 대칭수(갈래·빛살 개수) |
| 달 위상 | 전체 밝기·헤일로 |
| MBTI | 밀도·속도·반짝임·질서(T/F) |
| 생일 전체 | 개인 시드 → 입자 배치·hue 지터 고정 |

→ 이전엔 시각 축이 사실상 '오행 하나'라 같은 오행끼리 겹쳤음. 이제 조합이 사실상 1인 1수호신.

### 2-4. 이번에 제거한 것
- **휴먼디자인**: 온보딩 자가선택 UI·프로필 주입·프롬프트 지표 전부 삭제. 생일로 뽑히는 값을 손으로 고르게 한 게 세계관 위반이라 판단. (정식 자동계산은 Phase 2로 보류.)

## 4. 검증 상태 (중요 — 사실 그대로)

### ✅ 검증됨
- **문법:** esbuild 트랜스파일 통과, 에러 0 (v14·v15 모두).
- **시각 고유성:** 실제 코드 헬퍼로 6인 시뮬레이션 → **6/6 고유, 충돌 0**. 같은 오행(수) 3명이 주색·대칭까지 전부 갈라짐을 확인.

### ⚠️ 미확인 (실기기·실API 테스트 필요)
- **실제 판결 동작:** 아티팩트 환경에서 콜1/콜2가 정상 응답하는지.
- **프롬프트 캐싱 실효:** 아티팩트 프록시가 `cache_control`을 통과시키는지. (통과 안 돼도 에러 없이 캐싱만 미적용 — 안전.)
- **체감 속도:** 2번째 질문이 실제로 빨라지는지.
- **이어묻기:** "방금 말고 다른 선택은?"이 실제로 맥락을 잇는지.
- **모바일:** 과거 세션에서 모바일 API가 "Invalid response format"로 실패. **PC 크롬이 확인된 테스트 환경.**

## 8. 유지해야 할 핵심 원리

- **"수호신은 이미 너를 안다"** — 자동 파생 가능한 값은 손입력으로 되묻지 않는다(HD 제거 근거).
- **독립 시각축 원칙** — 지표를 서로 다른 지각 축에 배치해야 다양성이 살아난다(축이 겹치면 버킷 몇 개로 붕괴).
- **2콜 판결의 일관성** — 콜1이 결론을 확정하고 콜2는 그 결론의 근거만 설명. 이 순서를 뒤집으면 결론↔근거가 갈릴 수 있음.
- **경험 편향 규칙** — 지표가 팽팽하면 더 대담한·행동하는 선택 쪽으로.
