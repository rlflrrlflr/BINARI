# 비나리 기획전략 — PM 전략분석 (v15 코드 기준)

> 8에이전트 워크플로 교차검증(코드 898줄 라인단위 실사 + 한국 운세시장·행동심리 웹리서치 + 4렌즈 분석 + 적대적 비평)의 종합. 적대적 비평에서 기각된 주장(발명된 KPI 수치, 푸시 인프라 없는 알림 서사, 아티팩트 localStorage 오판 등 9건)은 제거하고 살아남은 결론만 담는다.

**분석 전제 한 줄:** v15는 "제품"이 아니라 **Claude 아티팩트 창에서만 살아있는 데모**다. fetch에 API 키가 없고(448행), 저장이 리포 전체 0건, 지인에게 URL을 보내면 90%는 폰으로 여는데 폰은 미확인.

## 1. 페르소나 — 서지우, 29세 마케팅팀 대리(INFP)
운세앱 실사용자 데이터 정합: 포스텔러 이용자 80%+가 2030, Z세대 42.9%가 '자기 파악' 목적 소비. 핵심 성향은 결정을 못 하는 게 아니라 **미룬다** — 결론은 마음에 있는데 실행할 '허가'가 없다. 친구에게 물으면 판단·마음의 빚, 혼자 정하면 책임이 무겁다.

**비포(화요일):** 07:40 옷 회피, 11:50 점심 "아무거나", 14:30 커피챗 답장 창 세 번 열었다 닫음, 19:40 헬스 내일로, 22:30 코트 15분 노려보다 인스타 도피. 이 머뭇거림마다 **어떤 앱도 안 열린다.**

**킬러 서비스 정의:** "답장을 쓰다 지우는 그 4초의 머뭇거림마다 카톡 대신 열려서, 3초 안에 '나의 서사'로 결론을 내려주고, 다음 날 그 이행을 보고하러 스스로 돌아오게 만드는 개인 판결소." 비나리가 차지할 슬롯은 운세앱의 아침이 아니라 **'쓰다 지운 카톡'의 슬롯**이다.

## 2. 현실 마찰 — 구조적 결론
> 비나리가 이기려던 판은 "룰렛(서사 없음)과 운세앱(내 얘기 아님) 사이"인데, 현재 코드는 **룰렛보다 느리고 운세앱보다 기억력이 나쁘다. 양쪽 최악만 조합.**

경쟁 슬롯: 지하철 이직→네이버 5초 / 회의 전→ChatGPT 20초(GPT사주 검색 129%↑) / 밤 11시 전남친→친구 카톡·헬로우봇(폰+푸시) / 자랑→심리테스트 링크 / 점심→룰렛 3초. 매 순간 비나리가 지는 이유는 부재·기억상실·공유불가·6단계 의식.

## 3. 4대 배관 공사 (최우선, 이거 전엔 전부 공중누각)
1. **저장**(localStorage) — 매 방문 2~4분 온보딩 반복이 '매일 쓰는 앱'의 1번 차단요인
2. **에러 복구**(catch 리셋) — 콜1 실패 시 재시도 버튼 없어 새로고침=온보딩 재수행 데드엔드
3. **내보내기**(toDataURL) — 유일한 바이럴 자산(부적)이 코드 0줄로 감옥
4. **API 프록시** — 키 없는 fetch라 아티팩트 밖에선 판결 100% 실패

## 4. 기능 판정 요약
- **살릴 것**: 2콜 판결 엔진, 3층 리빌, 토정비결(신년 시즌 자산)
- **고칠 것**: 파라메트릭 수호신(저장 없인 스크린세이버), 주역 의식(C형까지 강제 금지→선택 관문), 부적(내보내기 최우선), 바이오리듬(UI 승격=공짜 데일리 훅)
- **버릴 것**: 가짜 '건너뛰기', HD 죽은 코드, 첫 방문 강제 13탭(저장되면 기기당 1회라 완화)

## 5. 로드맵
- **0단계(2주)**: Vercel 서버리스 배포, 키 은닉, 데드엔드 수리, 모바일 완주. Supabase는 넣지 마라. **Anthropic 콘솔 월 지출 한도 먼저.**
- **1단계(1~2개월)**: localStorage→데일리 카드→부적 공유→스트릭. 순서가 전부(각 항목이 앞 없이는 무의미). 계정 금지.
- **2단계(3~6개월)**: 심층 풀이 건별(포스텔러 공식)→부적 소장판→구독. **송금 링크+수동 이행**으로 PG 없이 결제의사 측정. 이때 Supabase 도입.
