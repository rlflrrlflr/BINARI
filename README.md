# 비나리 (BINARI)

> 생년월일·성격 데이터로 '수호신'을 조립하고, 사용자의 고민에 AI가 **판결**을 내려주는 의사결정 지원 웹앱.

- **정체:** 브라우저에서 도는 React 웹앱(단일 JSX 아티팩트).
- **세계관:** "수호신은 사라진 게 아니라 흩어졌다" — 흩어진 조각(생일·성격 데이터)을 모아 나만의 수호신을 복원. 핵심 제약은 **"수호신은 이미 너를 안다"**(자동 파생 가능한 값은 손으로 되묻지 않는다).
- **흐름:** 온보딩(생일·MBTI·혈액형) → 파라메트릭 수호신 생성 → 질문 → AI 판결(3층 공개).
- **현재:** v26 — 동화 도입부·조각 보태기까지 완료, 자동 테스트 55종+렌더 검수 통과. 실제 API 동작(실기기)은 Vercel 배포 후 검증 예정.

자세한 배경·상태·다음 과제는 [`01_기획전략/비나리_기획전략_개발인계서_v15.md`](./01_기획전략/비나리_기획전략_개발인계서_v15.md) 참고.

이 저장소는 BINARI 프로젝트의 **백업/버전관리 저장소**입니다. 모든 산출물은 아래 폴더 구조와 파일명 규칙에 따라 정리·보관합니다.

> 문서 지도: [`변경이력.md`](./변경이력.md)(무엇을 바꿨나·한 줄) · [`작업로그.md`](./작업로그.md)(왜 그렇게 판단했나·서술) · [`app/ROADMAP.md`](./app/ROADMAP.md)(개발 진행 체크리스트)

## 📝 파일명 규칙

```
비나리_{영역}_{산출물}_vNN.ext
```

- **버전(`vNN`)은 항상 파일명 맨 뒤**(확장자 바로 앞)에 붙입니다.
- `{영역}`은 폴더 성격과 일치시킵니다: `기획전략`, `프롬프트엔진`, `비주얼프로토타입`, `리서치`.
- `{산출물}`은 문서의 구체적 이름입니다. 공백 없이 자연스럽게 작성합니다.
- 버전은 2자리(`v01`, `v02`, … `v15`)로 표기합니다.

### 예시

| 폴더 | 파일명 예시 |
|------|-------------|
| `01_기획전략/` | `비나리_기획전략_개발인계서_v15.md` |
| `02_프롬프트엔진/` | `비나리_프롬프트엔진_시스템프롬프트_v15.md` |
| `03_비주얼프로토타입/` | `비나리_비주얼프로토타입_MVP_v15.jsx` |
| `04_리서치/` | `비나리_리서치_시장조사_v15.pdf` |
| `09_구버전/` | `비나리_비주얼프로토타입_MVP_v14.jsx` |

## 🔁 버전 관리 원칙

1. **최신본**은 각 영역 폴더(`01`~`04`)에 둡니다.
2. 새 버전을 올릴 때, **직전 버전 파일은 `09_구버전/`으로 이동**합니다.
3. 파일명 규칙에서 버전 번호(`vNN`)만 올려서 이력을 추적합니다.
4. 모든 변경은 [`변경이력.md`](./변경이력.md)에 **날짜·버전·무엇·왜** 한 줄로 누적 기록합니다.
